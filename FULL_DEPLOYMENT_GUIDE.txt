### 1.2 Create .env.example Files (For Reference)

#### Backend (`backend/.env.example`)
```
# Database
MONGO_URI=mongodb+srv://username:password@cluster.mongodb.net/database

# JWT Secret
JWT_SECRET=your_super_secret_jwt_key_here

# Server
PORT=5000
NODE_ENV=development

# Cloudinary (Optional)
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
```

#### Frontend (`frontend/.env.example`)
```
VITE_API_URL=http://localhost:5000/api
```

### 1.3 Initialize Git and Push

```bash
# Go to project root
cd c:\Users\SHAYAN J\Desktop\WIndsurf-project

# Initialize git
git init

# Add all files except those in .gitignore
git add .

# Check status to verify .env files are not staged
git status

# First commit
git commit -m "Initial commit - Full MERN Stack E-commerce App"

# Create GitHub repository (go to github.com/new)
# Repository name: windsurf-gear
# Make it public or private

# Add remote
git remote add origin https://github.com/YOUR_USERNAME/windsurf-gear.git

# Push to GitHub
git push -u origin main

# If you get "error: failed to push", try:
git push -u origin master
```

### 1.4 Verify .env is NOT on GitHub

1. Go to your GitHub repo
2. Check `backend/` folder - `.env` should NOT be there
3. Only `.env.example` should be visible
4. If `.env` is there, remove it:
   ```bash
   git rm --cached backend/.env
   git commit -m "Remove .env from tracking"
   git push
   ```

## Step 2: Deploy Backend to Railway

### 2.1 Setup Railway Account

1. Go to https://railway.app/
2. Sign up with GitHub account
3. Click "New Project"
4. Select "Deploy from GitHub repo"

### 2.2 Deploy Backend

1. Select your `windsurf-gear` repository
2. Railway will auto-detect it's a Node.js app
3. Add Environment Variables in Railway Dashboard:
   
   Click on your service → Variables → New Variable
   
   ```
   MONGO_URI=mongodb+srv://your_user:your_pass@cluster.mongodb.net/windsurf-gear
   JWT_SECRET=your_super_secret_key_min_32_characters_long
   NODE_ENV=production
   PORT=3000
   CLOUDINARY_CLOUD_NAME=your_cloud_name
   CLOUDINARY_API_KEY=your_api_key
   CLOUDINARY_API_SECRET=your_api_secret
   ```

4. Railway will auto-deploy on every git push

### 2.3 Get Backend URL

1. Go to your service in Railway dashboard
2. Click "Settings" tab
3. Copy the "Domain" (e.g., `https://windsurf-gear.up.railway.app`)
4. Save this URL - you'll need it for frontend

### 2.4 Update Backend CORS

In `backend/server.js`, update CORS:

```javascript
const corsOptions = {
  origin: [
    'http://localhost:5173',  // Local development
    'https://your-vercel-frontend.vercel.app',  // Vercel frontend (will add later)
  ],
  credentials: true,
  optionsSuccessStatus: 200
};
app.use(cors(corsOptions));
```

Commit and push:
```bash
git add .
git commit -m "Update CORS for Railway deployment"
git push
```

## Step 3: Deploy Frontend to Vercel

### 3.1 Setup Vercel

1. Go to https://vercel.com/
2. Sign up with GitHub account
3. Click "Add New Project"
4. Import your `windsurf-gear` repository

### 3.2 Configure Build Settings

1. Framework Preset: Select "Vite"
2. Root Directory: `frontend` (IMPORTANT!)
3. Build Command: `npm run build`
4. Output Directory: `dist`

### 3.3 Add Environment Variables

Click "Environment Variables" and add:

```
VITE_API_URL=https://your-railway-backend.up.railway.app/api
```

Replace with your actual Railway backend URL.

### 3.4 Deploy

Click "Deploy" and wait for build to complete.

### 3.5 Get Frontend URL

1. After deployment, Vercel provides a URL
2. Example: `https://windsurf-gear.vercel.app`
3. Save this URL

---

## Step 4: Connect Frontend to Backend

### 4.1 Update Backend CORS with Frontend URL

Go back to Railway dashboard:
1. Select your backend service
2. Click "Variables"
3. No need to add, but update your `server.js`:

```javascript
const corsOptions = {
  origin: [
    'http://localhost:5173',
    'https://windsurf-gear.vercel.app',  // Your Vercel URL
  ],
  credentials: true,
  optionsSuccessStatus: 200
};
```

Commit and push - Railway will auto-redeploy.

### 4.2 Update Frontend API URL (If Changed)

If needed, update in Vercel:
1. Go to project settings
2. Environment Variables
3. Update `VITE_API_URL` if Railway URL changed

## Step 6: Custom Domain (Optional)

### Vercel Custom Domain

1. Buy domain (Namecheap, GoDaddy, etc.)
2. In Vercel: Project Settings → Domains
3. Add your domain
4. Update DNS at your registrar:
   - Type: CNAME
   - Name: www
   - Value: cname.vercel-dns.com

### Railway Custom Domain

1. In Railway: Service Settings → Domains
2. Click "Generate Domain" or add custom
3. If custom, add CNAME record pointing to Railway

## Troubleshooting

### Backend Not Connecting

1. Check Railway logs for errors
2. Verify `MONGO_URI` is correct
3. Check IP whitelist in MongoDB Atlas (add 0.0.0.0/0)
4. Verify `JWT_SECRET` is set

### Frontend API Errors

1. Check browser console for CORS errors
2. Verify `VITE_API_URL` matches Railway URL exactly
3. Check Network tab for failed requests
4. Ensure backend CORS includes Vercel domain

### Images Not Loading

1. Check if using Cloudinary (recommended)
2. Verify Cloudinary credentials in Railway
3. Check image URLs in database

### Admin Panel Not Working

1. Verify user role is "admin" in MongoDB
2. Check token is being sent in requests
3. Verify admin routes are protected
