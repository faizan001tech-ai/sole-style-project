# Windsurf Gear - MERN Stack E-Commerce Application

## Project Overview
Build a full-stack e-commerce application for watersports equipment using the MERN stack (MongoDB, Express, React, Node.js).

## Core Features Required

### Frontend (React + Vite)
1. **Modern UI/UX**
   - Glassmorphism design with premium animations using GSAP
   - Responsive mobile-first design with Tailwind CSS
   - Dark/Light mode support
   - Smooth transitions and scroll-triggered animations

2. **Pages**
   - Home page with Hero section, Featured Products, Best Sellers, Testimonials, Promo Banner
   - Shop page with product filtering and search
   - Product Details page with reviews
   - Cart page with quantity management
   - Checkout page with Cash on Delivery option
   - Login/Register pages with JWT auth
   - User Dashboard with order history and profile management
   - Admin Panel for product/order/user management
   - Contact page with working contact form
   - Support pages (FAQs, Shipping, Returns, Size Guide)
   - Company pages (About, Careers, Press, Sustainability)

3. **Components**
   - Navbar with cart icon and user menu
   - Footer with functional links
   - ProductCard with image, price, add to cart
   - ProtectedRoute and AdminRoute for authorization

### Backend (Node.js + Express)
1. **Authentication**
   - JWT-based authentication
   - User registration/login
   - Role-based access control (user/admin)
   - Password hashing with bcrypt

2. **API Endpoints**
   - Auth: /api/auth (register, login, profile, update profile)
   - Products: /api/products (CRUD, filters, featured, bestsellers, reviews)
   - Cart: /api/cart (get, add, update, delete, clear)
   - Orders: /api/orders (create, get my orders, get all orders, update status)
   - Users: /api/users (admin only - get all, update, delete, stats)
   - Contact: /api/contact (submit message, admin view/manage messages)

3. **Database Models**
   - User (name, email, password, role, avatar)
   - Product (name, description, price, category, image, stock, rating, reviews)
   - Cart (userId, items array with product and quantity)
   - Order (userId, items, total, status, shipping address, payment method)
   - ContactMessage (name, email, subject, message, status)

## Tech Stack

### Frontend
- React 18 with Vite
- Tailwind CSS for styling
- GSAP for animations
- React Router DOM for routing
- Axios for API calls

### Backend
- Node.js

## Key Implementation Details

### Authentication Flow
1. User registers/logs in
2. Backend returns JWT token
3. Frontend stores token in localStorage
4. Token sent with every protected request via Authorization header
5. Middleware verifies token and attaches user to request

### Contact Form Feature
- Public POST endpoint to submit messages
- Admin-only endpoints to view, update status, and delete messages
- Frontend form with validation and success/error feedback
- Admin Panel section showing all messages with filtering

### Cart Implementation
- Cart stored in database per user
- Add/remove/update quantities
- Calculate totals automatically
- Checkout creates order and clears cart

### Admin Panel Features
- Dashboard with stats (users, orders, products, revenue)
- Product management (add, edit, delete)
- Order management (view, update status)
- User management (view, update, delete)
- Contact messages (view, reply, delete)

### UI/UX Requirements
- Glassmorphism design with backdrop blur effects
- Dark/Light mode toggle
- Responsive design for all screen sizes
- Loading states and skeleton screens
- Toast notifications for user feedback
- Smooth animations with GSAP

## Environment Variables

### Backend (.env)
```
MONGO_URI=mongodb+srv://<username>:<password>@cluster.mongodb.net/windsurf-gear
JWT_SECRET=your_super_secret_jwt_key
PORT=5000
```

### Frontend (.env)
```
VITE_API_URL=http://localhost:5000/api
```

## Setup Instructions

1. **Backend Setup**
   ```bash
   cd backend
   npm install
   # Create .env file with MONGO_URI and JWT_SECRET
   npm start
   ```

2. **Frontend Setup**
   ```bash
   cd frontend
   npm install
   npm run dev
   ```

3. **Create Admin User**
   - Register a user
   - Manually update role to "admin" in MongoDB Atlas

## API Summary

- `POST /api/auth/register` - Register new user
- `POST /api/auth/login` - Login user
- `GET /api/auth/profile` - Get user profile (protected)
- `PUT /api/auth/profile` - Update profile (protected)
- `GET /api/products` - Get all products with filters
- `GET /api/products/featured` - Get featured products
- `GET /api/products/bestsellers` - Get bestsellers
- `GET /api/products/:id` - Get single product
- `POST /api/products` - Create product (admin)
- `PUT /api/products/:id` - Update product (admin)
- `DELETE /api/products/:id` - Delete product (admin)
- `POST /api/products/:id/reviews` - Add review (protected)
- `GET /api/cart` - Get user's cart (protected)
- `POST /api/cart` - Add item to cart (protected)
- `PUT /api/cart/:productId` - Update cart item (protected)
- `DELETE /api/cart/:productId` - Remove item from cart (protected)
- `DELETE /api/cart` - Clear cart (protected)
- `GET /api/orders` - Get all orders (admin)
- `GET /api/orders/myorders` - Get user's orders (protected)
- `GET /api/orders/:id` - Get order by ID (protected)
- `POST /api/orders` - Create order (protected)
- `PUT /api/orders/:id/status` - Update order status (admin)
- `PUT /api/orders/:id/pay` - Mark order as paid (protected)
- `GET /api/users` - Get all users (admin)
- `GET /api/users/:id` - Get user by ID (admin)
- `PUT /api/users/:id` - Update user (admin)
- `DELETE /api/users/:id` - Delete user (admin)
- `POST /api/contact` - Submit contact message (public)
- `GET /api/contact` - Get all messages (admin)
- `GET /api/contact/:id` - Get single message (admin)
- `PUT /api/contact/:id` - Update message status (admin)
- `DELETE /api/contact/:id` - Delete message (admin)
