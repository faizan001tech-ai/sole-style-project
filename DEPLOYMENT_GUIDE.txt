# Windsurf Gear - Deployment Guide

## Step 1: MongoDB Atlas Setup

1. Go to https://www.mongodb.com/atlas and create free account
2. Create new cluster (M0 Free Tier)
3. Click "Database Access" → Add New Database User
   - Username: your_username
   - Password: your_password
   - Role: Read and Write to Any Database
4. Click "Network Access" → Add IP Address
   - Select "Allow Access from Anywhere" (0.0.0.0/0)
5. Click "Databases" → "Connect" → "Connect your application"
6. Copy connection string:
   ```
   mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/windsurf-gear?retryWrites=true&w=majority
   ```

## Step 2: Backend Deployment (Render - Recommended)

### Option A: Render (Free)

1. Go to https://render.com/ and create account
2. Create New → "Web Service"
3. Connect your GitHub repository
4. Configure:
   ```
   Name: windsurf-gear-api
   Runtime: Node
   Build Command: npm install
   Start Command: node server.js
   ```
5. Add Environment Variables:
   ```
   MONGO_URI=mongodb+srv://your_user:your_pass@cluster.mongodb.net/windsurf-gear
   JWT_SECRET=your_super_secret_key_here
   PORT=10000
   NODE_ENV=production
   ```
6. Click "Create Web Service"
7. Copy the deployed URL (e.g., https://windsurf-gear-api.onrender.com)

### Option B: Railway (Alternative)

1. Go to https://railway.app/ and create account
2. New Project → Deploy from GitHub repo
3. Add variables in Railway dashboard
4. Deploy automatically

### Option C: Heroku

```bash
# Install Heroku CLI
# Login
heroku login

# Create app
heroku create windsurf-gear-api

# Add MongoDB
heroku addons:create mongolab:sandbox

# Set environment variables
heroku config:set JWT_SECRET=your_secret_key
heroku config:set NODE_ENV=production

# Deploy
git push heroku main
```

## Step 3: Frontend Deployment (Netlify)

### Method A: Drag & Drop (Easiest)

1. First build locally:
   ```bash
   cd frontend
   npm install
   npm run build
   ```

2. Go to https://www.netlify.com/
3. Login → Sites → Drag & Drop
4. Drag the `frontend/dist` folder
5. Site will be live instantly

### Method B: Git Integration (Recommended)

1. Push code to GitHub
2. Go to https://app.netlify.com/
3. "Add New Site" → "Import from Git"
4. Select your repository
5. Configure:
   ```
   Build Command: npm run build
   Publish Directory: dist
   ```
6. Add Environment Variables:
   ```
   VITE_API_URL=https://your-backend-url.onrender.com/api
   ```
7. Click "Deploy Site"

### Method C: Netlify CLI

```bash
# Install Netlify CLI
npm install -g netlify-cli

# Login
netlify login

# Deploy
cd frontend
netlify deploy --prod --dir=dist
```

## Step 4: Update CORS & API URLs

### Backend Update (IMPORTANT)

Update `backend/server.js` to allow your frontend domain:

```javascript
// CORS configuration
const corsOptions = {
  origin: process.env.NODE_ENV === 'production' 
    ? ['https://your-frontend-site.netlify.app', 'https://your-custom-domain.com']
    : 'http://localhost:5173',
  credentials: true,
  optionsSuccessStatus: 200
};
app.use(cors(corsOptions));
```

### Frontend Update

Create `frontend/.env.production`:
```
VITE_API_URL=https://your-backend-url.onrender.com/api
```

Or update hardcoded URLs in:
- `Contact.jsx`
- `AdminPanel.jsx`

Find and replace:
```javascript
// Change this:
'http://localhost:5000/api/...'

// To this:
import.meta.env.VITE_API_URL || 'https://your-backend-url.onrender.com/api'
```

### Netlify SPA Configuration

Create `frontend/public/_redirects` file:
```
/*    /index.html   200
```

This ensures React Router works on refresh.

### Vercel Configuration (Alternative to Netlify)

Create `vercel.json` in frontend root:
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

Deploy:
```bash
npm i -g vercel
vercel
```

## Common Issues & Solutions

### 1. CORS Error
```
Access to XMLHttpRequest blocked by CORS policy
```
**Solution:** Update backend CORS origin to match frontend domain exactly

### 2. MongoDB Connection Timeout
```
MongooseServerSelectionError: connection timed out
```
**Solution:** 
- Check IP whitelist in MongoDB Atlas (add 0.0.0.0/0)
- Verify connection string password (URL encode special characters)

### 3. JWT Token Issues
```
Error: invalid token
```
**Solution:** Ensure JWT_SECRET is same in both dev and production

### 4. 404 on Page Refresh (Frontend)
**Solution:** Add `_redirects` file for Netlify or `vercel.json` for Vercel

### 5. Images Not Loading
**Solution:** 
- Use Cloudinary/AWS S3 for images in production
- Or store images in backend `public` folder with static serving
